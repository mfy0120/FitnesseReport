Test Design Strategy


From above, we can define 5 different kinds of days:
a1. Day 1 to 27
a2. Day 28
a3. Day 29
a4. Day 30
a5. Day 31

4 kinds of months:
b1. Big month(contain 31 days):1, 3, 5, 7, 8, 10
b2. Small month(contain 30 days):4, 6, 9, 11
b3. December: 12
b4. February: 2

4 kinds of years:
c1. year % 4 != 0
c2. year % 4 == 0 && year % 100 != 0
c3. year % 100 == 0 && year % 400 != 0
c4. year % 400 == 0

We can make the leap/non-leap year, month, date as input, valid date, another month, another year as output.(Note: another month contains the date set to 1, another year contains another month and month set to 1).

So we should first test whether the function can judge the leap year correctly.

Results and Interpretation


From the tests above, we can see the program can executed correctly.
1. The reason I take the invalid date(see the third test) separate from test 2 is to minimize the original size of decision table. Because the date is invalid if there's at least one invalid date/month/year, so we can take this apart.
2. In class, Prof Hakan see we can consider Day 1 is different from day 2~27. Actually by using decision table method, this situation can be merged into day2~27. So we don't need to consider it as a different case.
3. There's one test case should be mentioned: If the current day is the last valid day(12/31/2200), the next day should be invalid(because the year exceed the limit)
